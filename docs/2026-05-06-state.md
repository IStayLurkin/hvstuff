# dayzdriv — State as of 2026-05-06

## What this is

Bare-metal VMX hypervisor driver for Windows 11 (i9-14900K, 32 logical cores).
Windows runs as the guest indefinitely. Driver loads via `sc create/start`.

- Driver: `bin\dayzdriv.sys` — the hypervisor
- Payload: `bin\payload\payload.sys` — manually-loaded guest kernel module (Ophion)
- Python bridge: `spectre\` — PyQt5 overlay + kernel comms via DeviceIoControl
- Build: double-click `build_dayz.bat` from Explorer (self-relaunches in VS Dev CMD)

## Current state — CLEAN BUILD, NOT YET HARDWARE-TESTED

Last two commits (2026-05-06):

### `887de78` — stealth revision + BSOD #12 EPT fix

**BSOD #12 fix (Ept.c — EptBuildIdentityMap):**
- Old code rounded each physical range outward to 2MB alignment, mapping PA holes
  between RAM ranges (firmware-reserved MMIO, PCIe BARs). Those PAs have no PFN entry.
  Ophion's guest CR3 walk called MmGetVirtualForPhysical on one → got garbage VA
  0xfffffffffffffe00 → crashed (PAGE_FAULT_IN_NONPAGED_AREA 0x50).
- Fix: head/body/tail scheme — 4KB pages for partial edges, 2MB large pages only for
  fully-aligned interior. Added explicit UC 4KB mappings for local APIC (0xFEE00000),
  IOAPIC (0xFEC00000), PCIe ECAM (0xE0000000–0xF0000000).

**Stealth features (not yet hardware-tested):**
1. **Firmware-Locked MSR**: `HandleRdmsr(IA32_FEATURE_CONTROL)` clears Lock + SMX/non-SMX
   enable bits (bits [2:0]), sets Lock=1 only. Guest sees VMX hardware-capable but
   firmware-disabled — identical to BIOS-disabled VMX.
2. **CPUID cache** (`CalibrateCpuidCache`, runs before VMXON): captures bare-metal
   responses for HV range [0x40000000–0x4000000F] (16 leaves) and 8 beyond-max-leaf
   leaves, plus minimum CPUID exit TSC cost (64 samples). `HandleCpuid` returns cached
   values for those ranges verbatim.
3. **RDTSC trap**: after every CPUID exit, `CPU_BASED_RDTSC_EXITING` is armed.
   `HandleRdtsc` returns `hardware_tsc - CpuidExitCost`, disarms immediately.
   Eliminates CPUID+RDTSC timing side-channel.
4. **DR isolation** (Arch.asm): `AsmVmExitHandler` saves host DR0-DR3/DR6 on stack
   (48h frame), loads guest shadow into hardware before dispatch, flushes hardware back
   to shadow after dispatch, restores host DRs before VMRESUME.
5. **Mode-aware RIP advance**: `AdvanceGuestRip` checks `GUEST_CS_ACCESS_RIGHTS` L/D
   bits — truncates new_rip to 32 bits (compat mode) or 16 bits (real/16-bit protected).
6. **EPT violation refactor**: `EptHandleViolation` returns NONE/HANDLED/EXEC (was
   BOOLEAN). X fault on hidden page injects #PF(0) instead of redirecting to real HPA.
   MBEC user-execute fault: lazy-grants EPT_EXEC_USER instead of injecting #GP.

### `4750c08` — IPI-broadcast INVEPT fix

**Problem**: `EptInvalidate` (INVEPT single-context) only flushes the calling core.
After `WP_REGISTER` or any EPT PTE change, other 31 cores had stale TLB entries.

**Fix**: `g_InveptPending` (volatile LONG). Every path that modifies EPT PTEs calls
`EptInvalidate` locally then `InterlockedExchange(&g_InveptPending, 1)`. At the top of
`VmExitDispatch` (after fast-path early returns), each core checks the flag; if set,
calls `EptInvalidate` and clears it. Safe, idempotent, no IPI/IRQL constraints.

Affected sites: `HV_CALL_WP_REGISTER`, `HV_CALL_SET_EPT_POLICY`,
`HandleEptViolation` (shadow HANDLED, MBEC lazy-grant, lazy UC map).

## Architecture quick-ref

| Layer | What it does |
|---|---|
| `VmxInitialize` | PASSIVE_LEVEL entry: calibrate CPUID, probe cores, launch resident HV via KeIpiGenericCall |
| `VmxLaunchCore` | IPI callback (IPI_LEVEL): VMXON → VMCS setup → AsmLaunchAndReturn → VMLAUNCH |
| `AsmVmExitHandler` | Save GPRs + host DRs, read exit reason, call VmExitDispatch, restore DRs, VMRESUME |
| `VmExitDispatch` | Check g_InveptPending, dispatch by exit reason, update TSC_OFFSET |
| `EptBuildIdentityMap` | head/body/tail 4KB+2MB identity map; explicit UC for MMIO holes |
| `EptHandleViolation` | Returns NONE/HANDLED/EXEC; R/W → decoy PTE; X → inject #PF(0) |

## Key invariants

- Never call ZwWriteFile or any pageable API at IPI_LEVEL. Log via HvLog only from
  DISPATCH_LEVEL or below (IPI callback uses pre-captured ctx fields instead).
- HOST_GDTR_BASE **and** GUEST_GDTR_BASE must both point at per-core ShadowGdt, never
  the real kernel GDT. CPU writes TSS busy bit into HOST_GDTR_BASE on every VM-exit.
- g_HostResumeRsp is saved AFTER `sub rsp,20h` — launch_resume must `add rsp,20h`
  before popping non-volatile registers.
- Never round EptBuildIdentityMap ranges outward past MmGetPhysicalMemoryRanges entries.
- Use __rdtsc() not KeQueryPerformanceCounter — the latter executes rdtscp which faults
  after vmx86.sys (VMware) leaves ENABLE_RDTSCP clear.
